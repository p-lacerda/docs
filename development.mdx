Este guia mostra como configurar seu ambiente local para desenvolvimento do **PediuAí**, usando nossa stack principal:

- .NET 8  
- Entity Framework Core  
- SQL Server  
- SignalR  
- WhatsApp Business API  
- Azure (hospedagem)  
- Docker  
- GitLab (CI/CD)

---

## Pré-requisitos

Antes de iniciar, certifique-se de ter o seguinte instalado:

- Node.js (v19 ou superior)
- Docker e Docker Compose
- .NET SDK 8
- Git
- Ferramentas de sua IDE (como VSCode ou Rider)

---

## Passo 1: Clonar o projeto

Clone o repositório principal hospedado no GitLab:

```bash
git clone https://gitlab.com/sua-org/api-pediuai.git
cd api-pediuai
```

---

## Passo 2: Configurar variáveis de ambiente

Copie o arquivo de exemplo `.env.example` para `.env` e edite os valores:

```bash
cp .env.example .env
```

Inclua suas configurações de banco de dados, tokens de API do WhatsApp e chaves de integração com o Azure.

---

## Passo 3: Subir ambiente com Docker

Execute o comando abaixo para iniciar o ambiente completo (API + SQL Server + dependências):

```bash
docker-compose up --build
```

---

## Passo 4: Aplicar Migrations

Após o ambiente estar rodando, aplique as migrations com Entity Framework:

```bash
dotnet ef database update
```

Se necessário, instale o EF CLI:

```bash
dotnet tool install --global dotnet-ef
```

---

## Passo 5: Acessar o sistema

Com tudo rodando, acesse o sistema localmente:

```
http://localhost:5000
```

ou conforme definido no seu `docker-compose.yml`.

---

## Dicas de desenvolvimento

- Use o SignalR para comunicação em tempo real com o painel e interfaces do cliente.
- Utilize tokens e webhooks da API do WhatsApp Business para integrar comunicação automatizada.
- Configure seu CI/CD com GitLab para deploy contínuo no Azure.
